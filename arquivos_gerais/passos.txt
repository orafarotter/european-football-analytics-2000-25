3. Clonar o repositorio para a sua estação

git clone <URL>

4. Acesse o arquivo terraform.tfvars.example em cd terraform/ e informe os valores das variáveis.
  4.1. informe os valores das variáveis 
  4.2. Copie o valor de 'composer_env_name' e salve em uma variável no terminal, como: export COMPOSER_ENV_NAME=<nome>
  4.3. Renomeie o arquivo para terraform.tfvars

5. Autentique-se no GCP 

gcloud auth application-default login

  5.1 Valide se está no projeto correto:

  gcloud config get-value project

  Caso não esteja...

  gcloud config set project <PROJECT_ID>

Depois execute:

gcloud auth application-default set-quota-project seu-project-id

6. Ainda no diretório cd terraform/ execute os comandos:

terraform init
terraform plan
terraform apply

Aguarde até o ambiente ser provisionado (pegue um café).

7. Dentro do GCP>IAM>Service Accounts, baixe a chave da conta de serviço que foi criada pelo Terraform

8. dbt Cloud 
  1. Crie um projeto no dbt Cloud;
  2. Utilize a conta de serviço recém criada para fazer a conexão com o BigQuery
  3. Estabeleça conexão com seu repositório e aponte para pasta 'football_dbt'
  4. No terminal do dbt execute o comando abaixo para instalar as dependencias
    dbt deps
  5. Criar o Environment de Produção
    1. Acesse Orchestration >> Environments
    2. + Create environment
      Environment settings
        - Environment name: Production
        - Environment type: Deployment
        - Set deployment type: Production
        - dbt version: Latest

      Connection profiles
        + Assign profile
        - Save

        + Assign profile

        Save






9. No seu terminal execute os seguintes comandos:

export KAGGLE_USERNAME=your_username
export KAGGLE_API_TOKEN=your_api_key

10. Faça o upload da DAG e das tasks para o Composer (talvez caiba um bash aqui)

gcloud composer environments storage dags import \
    --environment <COMPOSER_ENV_NAME> \ #look at terraform.tfvars
    --location us-east1 \
    --source dags/football_pipeline_dag.py

gcloud composer environments storage dags import \
    --environment <COMPOSER_ENV_NAME> \
    --location us-east1 \
    --source dags/scripts/

  10.1. Sete as variáveis do Kaggle no ambiente

gcloud composer environments run $ENV_NAME \
	--location us-east1 \
	variables set -- KAGGLE_USERNAME $KAGGLE_USERNAME

gcloud composer environments run $ENV_NAME \
	--location us-east1 \
	variables set -- KAGGLE_API_TOKEN $KAGGLE_API_TOKEN

  10.2. Suba a biblioteca do Kaggle

gcloud composer environments update $ENV_NAME \
    --location us-east1 \
    --update-pypi-packages-from-file requirements.txt
