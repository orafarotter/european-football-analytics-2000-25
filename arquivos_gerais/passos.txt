The dataset used in this project offers football match data from more than 30 leagues worldwide. The data spans from the 2000/01 season to the most recent results from the 2024/25 season. 
Link do dataset: https://www.kaggle.com/datasets/adamgbor/club-football-match-data-2000-2025

Our objective is discover between the top 10 European leagues, based in Opta Power Rankings in 02/04/2026, which leagues have:

- Total Matches by League
- Average Goals per Match by Year
- Match Result Distribution
- Match Intensity by League

Link do artigo da Opta: https://theanalyst.com/articles/strongest-football-leagues-in-the-world-opta-power-rankings

- Mencionar que é um ELT
- Mencionar que é usado apenas um arquivo do dataset

Comandos para reproduzir o projeto

1. git clone https://github.com/orafarotter/european-football-analytics-2000-25.git

cd european-football-analytics-2000-25

mv .env.example .env  

Preencha as variáveis do arquivo:

AIRFLOW_SECRET_KEY: comando python -c "import secrets; print(secrets.token_hex(32))"
AIRFLOW__CORE__FERNET_KEY: comando python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"

GCP_PROJECT_ID=your-gcp-project-id : Project ID do Google Cloud
GCP_REGION=your-gcp-region
GCS_BUCKET: your-gcs-bucket-name

KAGGLE_USERNAME=your_kaggle_username
KAGGLE_KEY=your_kaggle_api_key

comentário: 
KAGGLE_USERNAME: kaggle.com >> Settings >> Active logins >> Kaggle username
KAGGLE_KEY: kaggle.com >> Settings >> API Tokens >> Generate New Token

- - - 

Depois disso:

cd terraform

mv terraform.tfvars.example terraform.tfvars 

Preencha as variáveis do arquivo:

IMPORTANTE: project_id, region e bucket_name devem ser os mesmos que os informados em .env
Service acoount must match regex "^[a-z]([-a-z0-9]*[a-z0-9])?$"

project_id             = "<GCP_PROJECT_ID>"
region                 = "<REGION>"
bucket_name            = "<BUCKET_NAME>"
service_account_name   = "<SERVICE_ACCOUNT>"
