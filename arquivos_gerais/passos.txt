PRE REQUISITOS 

1. Ter uma conta no Google Cloud, uma billing account e um projeto

PROJECT_ID: euro-football-analytics-20-25
Number: 532542106535

2. Ter o Terraform instalado
https://developer.hashicorp.com/terraform/install


3. Ter uma conta no Kaggle.
Tenha seu username e token

Settings >> API Tokens >> Generate New Token

USERNAME
KEY

----------------

PASSOS

3. Clonar o repositorio para a sua estação

git clone <URL>

4. Acesse o arquivo terraform.tfvars.example em cd terraform/ e informe os valores das variáveis.
  4.1. Renomeie o arquivo para terraform.tfvars

5. Autentique-se no GCP 

gcloud auth application-default login

  5.1 Valide se está no projeto correto:

  gcloud config get-value project

  Caso não esteja...

  gcloud config set project <PROJECT_ID>

Depois execute:

gcloud auth application-default set-quota-project seu-project-id

6. Ainda no diretório cd terraform/ execute os comandos:

terraform init
terraform plan
terraform apply

Aguarde de 15 a 20 minutos para o ambiente ser provisionado

7. No seu terminal execute os seguintes comandos:

export KAGGLE_USERNAME=your_username
export KAGGLE_API_TOKEN=your_api_key

8. Faça o upload da DAG para o Composer

gcloud composer environments storage dags import \
    --environment <COMPOSER_ENV_NAME> \ #look at terraform.tfvars
    --location us-east1 \
    --source dags/football_pipeline_dag.py

gcloud composer environments storage dags import \
    --environment <COMPOSER_ENV_NAME> \
    --location us-east1 \
    --source dags/scripts/

  8.1. Sete as variáveis do Kaggle no ambiente

gcloud composer environments run $ENV_NAME \
	--location us-east1 \
	variables set -- KAGGLE_USERNAME $KAGGLE_USERNAME

gcloud composer environments run $ENV_NAME \
	--location us-east1 \
	variables set -- KAGGLE_API_TOKEN $KAGGLE_API_TOKEN
