## 🗺️ Pipeline Architecture

```
[Kaggle URL]
     │
     ▼ (Python via Airflow DAG — Docker)
[Cloud Storage — raw/matches.csv]   ← overwrite policy if file exists
     │
     ▼ (BigQuery External Table via Airflow DAG)
[BigQuery — dataset: eu_football_raw]
     │
     ▼ (dbt — triggered via Airflow DAG)
[BigQuery — dataset: eu_football_staging]   ← staging models (stg_)
     │
     ▼
[BigQuery — dataset: eu_football_mart]      ← mart models (partitioned + clustered)
     │
     ▼
[Looker Studio — Dashboard]
```

### Medallion Model
- **Bronze/Raw**: raw data in GCS + external table in BigQuery (`eu_football_raw`)
- **Silver/Staging**: cleaning, type casting, join with league seed (`eu_football_staging`)
- **Gold/Mart**: final aggregated table, partitioned by year and clustered by league, filtered for major European leagues (`eu_football_mart`)

**`stg_matches` model:**
- Type casting (`MatchDate` as DATE, goals as INTEGER, etc.)
- Join with `ref('leagues')` to bring in `country` and `league_name`
- Rename columns to snake_case
- Filter records where `MatchDate IS NOT NULL`

**`fct_european_matches` model:**
- Filter by European leagues: `WHERE division IN ('E0', 'SP1', 'I1', 'D1', 'F1', 'B1', 'P1', 'E1', 'DEN', 'POL')`
- Partitioned: `PARTITION BY DATE_TRUNC(match_date, YEAR)`
- Clustered by: `division` and `league_name`
- Calculated columns: `total_goals`, `goal_difference`, `match_result_label`, etc.

**dbt tests (going the extra mile):**
- `not_null` on critical fields
- `accepted_values` for `FTResult` (H, A, D)
- `relationships` between models

## 🐛 Current Bug: `run_dbt` Task Failing

### Symptom
The `run_dbt` Airflow task fails with **exit code 2** and produces **zero output** from dbt — the process fails in ~1.2 seconds before dbt even starts.

### Root Cause (confirmed)
The `dbt` binary is installed at `/home/airflow/.local/bin/dbt` (pip user install), but this path is **not included in the PATH** inherited by the BashOperator subprocess. The shell cannot find the `dbt` command.

This was confirmed by running inside the scheduler container:
```bash
docker exec -it european-football-analytics-2000-25-airflow-scheduler-1 bash
which dbt           # returns empty — not in PATH
find /home/airflow -name "dbt" 2>/dev/null  # returns /home/airflow/.local/bin/dbt
/home/airflow/.local/bin/dbt --version      # works fine
```

### Current (broken) DAG task:
```python
DBT_PROJECT_DIR = "/opt/airflow/football_dbt"

t4_dbt_run = BashOperator(
    task_id="run_dbt",
    append_env=True,
    bash_command=(
        "dbt deps --project-dir {{ params.dbt_dir }} --profiles-dir {{ params.dbt_dir }} && "
        "dbt seed --project-dir {{ params.dbt_dir }} --profiles-dir {{ params.dbt_dir }} && "
        "dbt run --project-dir {{ params.dbt_dir }} --profiles-dir {{ params.dbt_dir }} && "
        "dbt test --project-dir {{ params.dbt_dir }} --profiles-dir {{ params.dbt_dir }}"
    ),
    params={"dbt_dir": DBT_PROJECT_DIR},
)
```

### Fix to Apply (NOT YET APPLIED)
Replace `dbt` with the absolute path `/home/airflow/.local/bin/dbt`:

```python
DBT_PROJECT_DIR = "/opt/airflow/football_dbt"
DBT_BIN = "/home/airflow/.local/bin/dbt"  # explicit path — avoids PATH issues

t4_dbt_run = BashOperator(
    task_id="run_dbt",
    append_env=True,
    bash_command=(
        "{{ params.dbt_bin }} deps  --project-dir {{ params.dbt_dir }} --profiles-dir {{ params.dbt_dir }} && "
        "{{ params.dbt_bin }} seed  --project-dir {{ params.dbt_dir }} --profiles-dir {{ params.dbt_dir }} && "
        "{{ params.dbt_bin }} run   --project-dir {{ params.dbt_dir }} --profiles-dir {{ params.dbt_dir }} && "
        "{{ params.dbt_bin }} test  --project-dir {{ params.dbt_dir }} --profiles-dir {{ params.dbt_dir }}"
    ),
    params={
        "dbt_dir": DBT_PROJECT_DIR,
        "dbt_bin": DBT_BIN,
    },
)
```
